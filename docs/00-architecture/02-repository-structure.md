# Repository Structure

## Top-Level Layout

- `src/`: package source code
- `tests/`: unit, integration, architecture, benchmark tests
- `docs/`: project documentation
- `config/`: package configuration for MongoDB-related logging
- `scripts/`: quality and release helper scripts
- `.github/workflows/`: CI/CD pipelines

## Notes

This structure is intentionally aligned with DTO documentation indexing while preserving client-specific runtime assets.
